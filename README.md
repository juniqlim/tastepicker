# tastepicker

내 취향에 맞는 큐레이터를 내가 고르고, 그가 고른 것을 본다.

평점과 랭킹은 다루지 않는다. 다수의 판단이 아니라 내가 신뢰하는 소수의 판단만 모은다.

## 원칙

- **픽커는 내가 임명한다.** 서비스는 후보를 추천할 수 있어도 목록에 자동으로 넣지 않는다.
- **출처를 밝힌다.** 모든 픽은 픽커와 원문 링크를 반드시 가진다.
- **원문은 픽커의 것으로 보여준다.** 서비스가 쓴 글처럼 내보이지 않는다. 인용은 발췌로 하고 원문으로 넘어갈 길을 남긴다.
- **많이보다 확실하게.** 근거가 불분명한 데이터는 버린다.

## 용어

| 용어 | 뜻 |
| --- | --- |
| picker | 내가 검증해서 등록한 큐레이터 |
| pick | 픽커가 고른 것 |

픽의 공통 항목은 `picker`와 `link` 둘뿐이다. 나머지는 픽커가 정한다.
맛집 픽커는 `region`·`name`을, 투자 픽커라면 다른 항목을 담는다.

## 현재

네이버 블로그 RSS에서 맛집 픽을 뽑는다. RSS는 최근 50글까지만 싣는다.

```
src/rss.js       RSS → 글 목록
src/pickers.js   글 목록 → 픽 (픽커마다 규칙 하나)
```

픽커마다 제목 형식이 달라 범용 파서를 두지 않는다. 소수만 다루므로 규칙을 하나씩 쓰는 쪽이 정확하다.

| 픽커 | 형식 | 결과 |
| --- | --- | --- |
| [정직한 청년](https://blog.naver.com/thddbcjf) | `지역 가게명-한줄평` | 38/38 |
| [RockHer](https://blog.naver.com/fascinoya) | `[지역 분류] 가게명 (등급) - 설명` | 17/30 |

RockHer는 본인이 `(강추)`·`(추천)` 등급을 붙인 글만 받는다. 등급 없는 글을 추천으로 볼 근거가 없다.
